Changelog

v2.22
---------------------------------
• If FFVship fails, Auto-Boost now tries the vs-hip fallback automatically.
  This can help an encode continue instead of stopping right away.
• Added a clear message when this happens:
  FFVship failed, using vs-hip fallback
• The GitHub repo no longer includes the full VapourSynth folder.
  The repo now only has VapourSynth\README.md.
  If you need the ready-to-use VapourSynth files, get them from a release zip.
• Added a small test to help make sure the FFVship fallback stays working in future updates.

v2.2
---------------------------------
• Restored and improved SSIMU2 metric backend support
  Auto-Boost can now use vs-hip/libvship_NVIDIA.dll, vs-hip/libvship_VULKAN.dll, FFVship NVIDIA, FFVship Vulkan, or vs-zip.
  Existing aliases such as vs-hip, libvship_NVIDIA.dll, libvship_VULKAN.dll, ffvship_nvidia, ffvship_vulkan, and vs-zip are supported.
• Improved workercount-ssimu2.txt behavior
  The config now supports key-based fields for tool, filter-tool, workercount, variant, and streams.
  If any VapourSynth filtering is enabled, filter-tool prefers vs-hip and falls back to vs-zip because FFVship cannot run filter graphs.
• Added dynamic SVT-AV1 fork selection
  Batch files and dispatch scripts can select 5fish, essential, hdr, or custom forks using --fork.
  Fork folders are matched by stable names like 5fish, essential, and hdr so updated folder versions do not require exact path edits.
  5fish and essential can use AVX-512 builds when --avx512 is set; hdr uses its x86-64-v3 build.
  Essential also copies ffms2.dll beside SvtAv1EncApp.exe when required.
• Updated bat-builder.py and existing batch files
  Generated and included .bat files now pass --fork, optional --avx512, and --denoise to the dispatchers.
  Auto-Boost batch files read workercount-ssimu2.txt by key instead of assuming fixed line positions.
  Default SSIMU2 fallback is vs-hip with one worker if no benchmark config exists.
• Added batch-driven denoise control
  dispatch.py and av1an-dispatch.py can set denoise=True or denoise=False in settings.txt before encoding.
  Auto-Boost dispatch also updates the temp copy of settings.txt so resumed/temp runs use the selected denoise value.
  bat-builder defaults denoise=False except for 5fish, where denoise=True is recommended with the DFTTest denoise graph.
• Fixed portable helper imports under bundled Windows Python
  dispatch.py, av1an-dispatch.py, and Auto-Boost-Av1an.py now add their tools folder to sys.path before importing svt_fork_setup.py.
• Fixed Auto-Boost fork handoff during encoding
  dispatch.py now forwards the selected fork and AVX-512 flag into Auto-Boost-Av1an.py so a 5fish batch file does not get overwritten by the default essential fork during the encode stage.
• Improved vs-hip DLL activation
  Auto-Boost no longer deletes/re-copies locked libvship DLLs from VapourSynth\vs-plugins while they are already loaded.
  If core.vship.SSIMULACRA2 is already available, the existing loaded plugin is treated as valid and encoding can continue.

v1.35
---------------------------------
• Automated Hardware Optimization: Added a one-time test that runs the first time you start a batch file to ultimately speed up final-pass av1 encoding.
  - It benchmarks your system's actual RAM usage and CPU threads to calculate the safest and fastest "Worker" count for your specific PC.
  - Results are saved to a configuration file (tools\workercount-config.txt) so the test only needs to run once. You may manually edit this txt file if needed.
  - You can safely delete the txt file tools\workercount-config.txt if you want the test to run again.
  - Optimized Threading: Automatically adds --lp 3 to final encoding settings. This improves efficiency and prevents CPU oversaturation when running multiple workers.
• New Troubleshooting Tool: Added extras\lossless-intermediary.bat.
  - Use this if you have a "problematic" video that crashes or refuses to encode properly.
  - It converts your video into a clean, lossless file that you can use as a stable source to ensure smooth encoding. Place your mkv file in the "tools" folder and
	run lossless-intermediary.bat
• New Opus Audio Tool: Added extras\encode-opus.audio.bat for easy audio compression.
  - Automatically extracts audio from your MKV videos and converts it to high-quality, space-saving Opus format.
  - Smartly processes multiple files at once using your CPU's full threads and saves a new video with opus audio to an output folder.
• Add python script to automatically cleanup all temp files and folders
